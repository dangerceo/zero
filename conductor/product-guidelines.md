# Product Guidelines: Zero Computer

## Design Principles
Zero Computer follows the **Modern/Material Design** aesthetic, prioritizing a consistent experience with the Android mobile app's Material 3 theme. The interface is clean, approachable, and leverages modern design patterns to bridge the gap between complex Mac automation and intuitive mobile control.

## UX Principles
- **Speed/Responsiveness:** All remote actions, agent status updates, and terminal feedback must be delivered instantly via WebSockets to ensure a seamless experience.
- **Accessibility/Clarity:** Layouts should be optimized for mobile use, providing clear, high-contrast indicators for system status and agent progress.

## Voice and Tone
- **Informative/Detailed:** Communication should be thorough and explain exactly what the system or AI agents are doing. This builds trust and helps users understand the automation process.
- **Transparent:** The system should never leave the user guessing; every action should have a corresponding status or feedback message.

## Error Handling and Feedback
- **Visual/Status-based:** Errors and updates should be communicated through clear visual cues and status-based indicators (e.g., color-coded icons, progress bars, and high-level summaries).
- **Graceful Recovery:** When an error occurs, the system should provide a clear path to recovery or a high-level explanation of the issue, avoiding obscure technical logs where possible unless requested.

## Branding
- **Logo/Iconography:** Use consistent icons across web and mobile that reflect the Material 3 design system.
- **Color Palette:** A modern, accessible color palette that works well in both light and dark modes, with secondary accents used for status and alerts.
